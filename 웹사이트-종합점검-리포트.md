## 1. 전체 개요

### 1.1 사이트 목적
- **비즈니스 모델**: B2B 인테리어/익스테리어 전문 서비스
- **타겟 고객**: 펜션, 대형카페, 카라반, 캠핑장 운영자 및 사업자
- **핵심 가치**: 특수 공간에 특화된 전문 인테리어 솔루션 제공

### 1.2 기술 스택
- **프론트엔드**: HTML5, CSS3, JavaScript (ES6+)
- **스타일링**: 커스텀 CSS (styles.css, page-styles.css, product-styles.css, admin-styles.css)
- **아이콘**: Font Awesome 6.0.0
- **폰트**: Inter, Playfair Display (Google Fonts)
- **데이터 저장**: JSON 파일 (products.json, portfolio.json) + LocalStorage

## 5. 데이터 관리 시스템

### 5.1 제품 데이터 구조

**파일**: `data/products.json`
```json
{
  "categories": {
    "outdoor": [...],
    "furniture": [...],
    "lighting": [...],
    "flooring": [...],
    "wall": [...],
    "accessories": [...]
  },
  "featured": [...]
}
```

**제품 필드**:
- id, name, category, subcategory
- price, stock, status
- description, shortDescription
- size, material, badge
- image, images (배열)
- features (배열)
- specs (객체)
- options (객체)
- popularity, updated

### 5.2 포트폴리오 데이터 구조

**파일**: `data/portfolio.json`
```json
{
  "projects": [
    {
      "id", "title", "category",
      "highlighted", "location", "area", "duration",
      "description", "tags", "cover",
      "products", "metrics"
    }
  ]
}
```

### 5.3 데이터 저장 방식

#### 현재 구조:
1. **JSON 파일**: 정적 데이터 소스
2. **LocalStorage**: 
   - 관리자에서 수정한 제품 데이터
   - 위시리스트
   - 문의 리드 데이터
   - 관리자 로그인 상태

#### 장점:
- 빠른 로딩
- 서버 없이 동작 가능
- 간단한 구조

#### 단점:
- 브라우저별 데이터 분리 (공유 불가)
- 데이터 영구 저장 불가
- 멀티 사용자 미지원
- 실제 운영 환경에 부적합

## 6. 보완이 필요한 사항

### 6.1 🔴 긴급 (P0) - 즉시 수정 필요

#### 6.1.1 데이터 영구 저장 문제
**현황**: 모든 데이터가 LocalStorage에만 저장되어 브라우저를 지우면 사라짐

**문제점**:
- 관리자가 제품을 추가/수정해도 다른 사용자에게 반영되지 않음
- 새 방문자는 기본 하드코딩된 제품만 볼 수 있음
- 실제 운영 환경에서 사용 불가

**해결 방안**:
- 백엔드 API 구축 (Node.js + Express + MongoDB/PostgreSQL)
- 또는 Headless CMS 도입 (Strapi, Contentful, Sanity 등)
- 또는 서버리스 솔루션 (Supabase, Firebase)

#### 6.1.2 관리자 인증 보안
**현황**: LocalStorage에 플래그만 저장, 비밀번호 평문 저장

**문제점**:
- 보안 취약
- 다중 계정 관리 불가
- 세션 만료 없음

**해결 방안**:
- JWT 토큰 기반 인증
- 비밀번호 해싱 (bcrypt)
- 세션 관리
- 권한 관리 (RBAC)

#### 6.1.3 존재하지 않는 페이지 링크
**현황**: 일부 메뉴에서 존재하지 않는 페이지로 링크

**문제점**:
- 404 에러 발생
- 사용자 경험 저하

**해결 방안**:
- 모든 링크 확인 및 수정
- 또는 누락된 페이지 생성

### 6.2 🟡 중요 (P1) - 단기 개선 필요

#### 6.1.4 실제 이미지/콘텐츠 확보
**현황**: 모든 이미지가 Unsplash 플레이스홀더

**문제점**:
- 저작권 이슈 가능성
- 실제 제품/프로젝트와 불일치
- 신뢰도 저하

**해결 방안**:
- 실제 제품 사진 촬영
- 실제 프로젝트 사진 확보
- 이미지 최적화 (WebP 변환)

#### 6.1.5 폼 처리 개선
**현황**: Formspree 연동은 되어 있으나 추가 처리 부족

**문제점**:
- 스팸 방지 없음
- 파일 첨부 불가 (도면 등)
- 자동 응답 이메일 없음

**해결 방안**:
- reCAPTCHA 추가
- 파일 업로드 기능
- 자동 응답 이메일 템플릿

#### 6.1.6 포트폴리오 상세 페이지 부재
**현황**: 포트폴리오는 갤러리 형태로만 표시

**문제점**:
- 프로젝트 상세 정보 부족
- Before/After 비교 없음
- 프로젝트별 제품 리스트 미표시

**해결 방안**:

**구현 완료도**: 약 85%
- 기본 기능은 모두 구현됨
- UI/UX는 현대적이고 사용자 친화적

- 프론트엔드는 완성도 높음
- 백엔드/데이터베이스 필요
- 실제 콘텐츠 확보 필요

### 권장 사항

1. **즉시 조치**:
   - 백엔드 API 또는 Headless CMS 도입
   - 실제 제품 이미지 확보
   - 존재하지 않는 페이지 링크 수정

2. **단기 개선**:
   - 관리자 인증 시스템 강화
   - 폼 처리 개선
   - 포트폴리오 상세 페이지

3. **중장기 발전**:
   - 멤버 포털 구축
   - 고급 기능 추가
   - 다국어 지원

이 두 가지를 해결하면 **즉시 서비스 가능한 수준**의 웹사이트가 됩니다.
